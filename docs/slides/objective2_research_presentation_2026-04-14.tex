\begin{frame}{Starting Point}
  \begin{itemize}
    \item The project started from a mathematical question, not from a trading rule.
    \item In class, we studied model selection using polynomial basis functions:
    \[
      1,\ x,\ x^2,\ \dots,\ x^d
    \]
    \item That raised the main idea:
    \[
      \text{Can trading strategies also be treated as basis functions?}
    \]
    \item If a strategy maps a market state into a numerical score, then it can be evaluated across dates and placed into a design matrix.
  \end{itemize}
\end{frame}

\begin{frame}{Function Space Analogy}
  \small
  \begin{tabular}{lll}
    \toprule
    Concept & Polynomial Model & Strategy Model \\
    \midrule
    basis function & \(1, x, x^2, \dots\) & strategy signal functions \\
    data point & \(x_i\) & market state at date \(t_i\) \\
    matrix entry & \(\phi_j(x_i)\) & strategy score \(s_j(t_i)\) \\
    model selection & choose degree / penalty & choose strategy basis / penalty \\
    target & observed \(y_i\) & future cumulative return \\
    \bottomrule
  \end{tabular}

  \vspace{0.8em}
  Core design matrix:
  \[
    A_{ij} = \phi_j(\text{market state at } t_i)
  \]

  \vspace{0.4em}
  Here, each column is a strategy function evaluated over time.
\end{frame}

\begin{frame}{Research Question}
  \begin{block}{Main Question}
    Can a matrix of strategy signals explain or predict future GLD returns?
  \end{block}

  \vspace{0.6em}
  \begin{itemize}
    \item First, define a strategy space numerically.
    \item Then test whether linear operators on that space have predictive value.
    \item Finally, translate predictions into portfolio exposure.
  \end{itemize}

  \vspace{0.5em}
  Important distinction:

  \[
    \text{building a valid strategy space}
    \neq
    \text{proving every linear combination is better}
  \]
\end{frame}

\begin{frame}{Four Initial Strategy Families}
  \begin{itemize}
    \item \textbf{Adaptive Band}
    \begin{itemize}
      \item price location relative to rolling normalized bands
    \end{itemize}
    \item \textbf{Moving-Average Crossover}
    \begin{itemize}
      \item trend state from short and long moving averages
    \end{itemize}
    \item \textbf{Adaptive Volatility Band}
    \begin{itemize}
      \item volatility state from high-low-close structure
    \end{itemize}
    \item \textbf{Fear-Greed Candle Volume}
    \begin{itemize}
      \item event-style signal from candle body, volume, and price zone
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Strategy 1: Adaptive Band}
  \begin{columns}[T]
    \begin{column}{0.40\textwidth}
      \small
      \textbf{Signal type: continuous}
      \vspace{0.5em}
      \begin{itemize}
        \item Price is normalized relative to a rolling moving average
        \item Upper and lower bands are placed at scaled thresholds around the MA
        \item Score reflects how far price sits from the band midpoint
        \item Buy zone: price near or below lower band
        \item Sell zone: price near or above upper band
      \end{itemize}
    \end{column}

  \end{columns}
\end{frame}

\begin{frame}{Strategy 2: Moving-Average Crossover}
  \begin{columns}[T]
    \begin{column}{0.40\textwidth}
      \small
      \textbf{Signal type: continuous}
      \vspace{0.5em}
      \begin{itemize}
        \item Compares a short-window MA against a long-window MA
        \item Positive spread indicates uptrend; negative spread indicates downtrend
        \item Score captures both direction and magnitude of the crossover gap
        \item Buy signal: short MA crosses above long MA
        \item Sell signal: short MA crosses below long MA
      \end{itemize}
    \end{column}
    \begin{column}{0.57\textwidth}

\end{frame}

\begin{frame}{Strategy 3: Adaptive Volatility Band}
  \begin{columns}[T]
    \begin{column}{0.40\textwidth}
      \small
      \textbf{Signal type: continuous}
      \vspace{0.5em}
      \begin{itemize}
        \item Uses a volatility proxy derived from the high-low-close range
        \item Bands are placed relative to the rolling volatility level
        \item Score reflects the current volatility regime position
        \item High-vol breakouts and low-vol compressions generate distinct signals
        \item Complementary to price-band strategies
      \end{itemize}
    \end{column}
    \begin{column}{0.57\textwidth}

\end{frame}

\begin{frame}{Strategy 4: Fear-Greed Candle-Volume}
  \begin{columns}[T]
    \begin{column}{0.40\textwidth}
      \small
      \textbf{Signal type: event-style (discrete)}
      \vspace{0.5em}
      \begin{itemize}
        \item Fires only when both candle body size and volume spike simultaneously above rolling thresholds
        \item Captures extreme sentiment: large directional move confirmed by high activity
        \item Rare by design: most days produce no signal
        \item Unlike the other three strategies, this is not a continuous score
        \item Adds a qualitatively different basis vector to the design matrix
      \end{itemize}
    \end{column}

  \end{columns}
\end{frame}

\begin{frame}{Objective 1: Return-Based Strategy Combination}
  \begin{itemize}
    \item Objective 1 combined realized strategy returns.
    \item For each anchor date:
    \begin{itemize}
      \item optimize candidate parameters inside each strategy family
      \item select representative candidates
      \item build a daily return matrix
      \item estimate portfolio weights
    \end{itemize}
    \item Weight constraints tested:
    \begin{itemize}
      \item signed weights
      \item long-only weights
    \end{itemize}
  \end{itemize}

  \vspace{0.6em}
  Objective 1 built the rolling optimization infrastructure.
\end{frame}

\begin{frame}{Why Objective 2 Was Needed}
  \begin{itemize}
    \item Objective 1 asks:
    \[
      \text{How should realized strategy returns be combined?}
    \]
    \item Objective 2 asks a different question:
    \[
      \text{Do current strategy states predict future returns?}
    \]
    \item This moves the project from portfolio mixing to supervised learning.
    \item The strategy outputs become features.
    \item Future cumulative return becomes the target.
  \end{itemize}
\end{frame}

\begin{frame}{Objective 2: Initial Design Matrix}
  \[
    \hat{y}_t = \beta_0 + A_t \beta
  \]

  \begin{itemize}
    \item Initial \(A\) matrix had four columns:
    \begin{itemize}
      \item adaptive band score
      \item MA crossover score
      \item adaptive volatility band score
      \item fear-greed score
    \end{itemize}
    \item Target:
    \[
      y_t^{(h)} = \frac{P_{t+h}}{P_t} - 1
    \]
    \item \(h\) is the future cumulative-return horizon.
  \end{itemize}
\end{frame}

\begin{frame}{Normal Equation Baseline}
  \begin{itemize}
    \item The first benchmark was ordinary least squares:
    \[
      \min_{\beta} \|y - A\beta\|^2
    \]
    \item The classical normal-equation form is:
    \[
      \hat{\beta} = (A^\top A)^{-1}A^\top y
    \]
    \item In code, this is solved with stable numerical least squares.
    \item This made the strategy-space idea directly comparable to standard linear supervised learning.
  \end{itemize}
\end{frame}

\begin{frame}{Horizon Scan}
  \begin{itemize}
    \item Instead of predicting only the next day, we scanned:
    \[
      h = 1, 2, \dots, 130
    \]
    \item The target is not the return on day \(h\).
    \item It is the full cumulative return from today to \(h\) trading days later:
    \[
      y_t^{(h)} = \frac{P_{t+h}}{P_t} - 1
    \]
    \item This asks which future time scale the strategy space naturally explains.
  \end{itemize}
\end{frame}

\begin{frame}{Expanded Strategy Space}
  \begin{itemize}
    \item The initial 4-column space was expanded to 40 basis columns.
    \item Construction:
    \begin{itemize}
      \item top 10 adaptive band candidates
      \item top 10 MA crossover candidates
      \item top 10 adaptive volatility candidates
      \item top 10 fear-greed candidates
    \end{itemize}
    \item This is closer to the class model-selection exercise:
    \[
      \text{many basis functions} \rightarrow \text{choose the useful structure}
    \]
    \item Models compared:
    \[
      \text{OLS, Ridge, Lasso, Elastic Net}
    \]
  \end{itemize}
\end{frame}

\begin{frame}{Forward Validation Design}
  \begin{itemize}
    \item For each anchor date, only the anchor's known optimization snapshot is used.
    \item Build the 40-column strategy matrix from that snapshot.
    \item Fit models over the selection period.
    \item Evaluate on the following future period.
    \item This separates:
    \begin{itemize}
      \item model discovery
      \item forward validation
      \item investment translation
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Interpretation of Horizon Shifts}
  \begin{itemize}
    \item Long horizons near 120--130 days appeared when GLD moved with slower, persistent drift.
    \item Medium horizons near 45--73 days appeared when transitions or acceleration mattered more.
    \item This suggests that the model is not only learning a strategy signal.
    \item It is also learning the natural time scale of the current market regime.
  \end{itemize}

  \vspace{0.6em}
  Practical interpretation:
  \[
    \text{slow trend regime} \rightarrow \text{long horizon}
  \]
  \[
    \text{transition regime} \rightarrow \text{shorter horizon}
  \]
\end{frame}

\begin{frame}{From Prediction to Investment}
  \begin{itemize}
    \item The model output is a predicted future cumulative return.
    \item It is converted into a long-only target weight:
    \[
      w_t = \operatorname{clip}\left(\frac{\hat{y}_t}{s}, 0, 1\right)
    \]
    \item Then the portfolio is split into rolling tranches.
    \item Each day:
    \begin{itemize}
      \item compute the signal
      \item allocate one tranche based on \(w_t\)
      \item hold the tranche for the fixed horizon
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Tranche Interpretation}
  \begin{itemize}
    \item A 45-day tranche is more responsive.
    \item A 130-day tranche is slower and more defensive.
    \item This translates the predictive horizon into an executable allocation rule.
  \end{itemize}

  \vspace{0.8em}
  \small
  \begin{tabular}{lll}
    \toprule
    Tranche & Behavior & Practical Role \\
    \midrule
    45-day & faster reaction & transition capture \\
    130-day & slower reaction & defensive long-horizon filter \\
    \bottomrule
  \end{tabular}
\end{frame}

\begin{frame}{Update Frequency Experiment}
  \begin{itemize}
    \item The next question was operational:
    \[
      \text{How often should the model be refreshed?}
    \]
    \item We compared:
    \[
      1M,\ 2M,\ 3M,\ 6M
    \]
    \item This tests the trade-off between:
    \begin{itemize}
      \item fast adaptation
      \item noise chasing
      \item regime stability
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Capital Efficiency View}
  \begin{itemize}
    \item Absolute return is not the only metric.
    \item In a multi-asset system, unused capital can be allocated elsewhere.
    \item So we also track exposure-adjusted efficiency:
    \[
      \text{100\%-Eq Return} \approx
      \frac{\text{Strategy Return}}{\text{Average Exposure}}
    \]
    \item Across 40 tested runs:
    \begin{itemize}
      \item raw return beat buy-and-hold in 10 cases
      \item efficiency proxy beat buy-and-hold in 17 cases
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{What Still Needs Work}
  \begin{itemize}
    \item The model is not yet a stable regime-aware exposure controller.
    \item Block analysis showed mixed results:
    \begin{itemize}
      \item sometimes exposure rose in good blocks
      \item sometimes it also rose in weak blocks
    \end{itemize}
    \item This suggests the next frontier is not only predicting return.
    \item It is learning when exposure itself should be high or low.
  \end{itemize}
\end{frame}

\begin{frame}{Live Deployment Direction}
  \begin{itemize}
    \item Deployment target: GLD through Alpaca paper trading.
    \item Daily flow:
    \begin{enumerate}
      \item update GLD daily data
      \item compute latest strategy-space signal
      \item build target position
      \item submit or log the order
      \item send email alert
    \end{enumerate}
    \item Raspberry Pi runs the daily pipeline with a systemd timer.
  \end{itemize}
\end{frame}

\begin{frame}{Final Takeaways}
  \begin{itemize}
    \item Strategy outputs can be treated as evaluated basis functions.
    \item This creates a valid numerical strategy space.
    \item Model selection on this space reveals horizon and regime structure.
    \item Tranche simulation translates prediction into allocation.
    \item The long-term goal is a multi-asset allocation system, not only a GLD strategy.
  \end{itemize}

  \vspace{0.8em}
  \centering
  \large
  Strategy-space modeling turns interpretable trading rules into a testable predictive geometry.
\end{frame}
