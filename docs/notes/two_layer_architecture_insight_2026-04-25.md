# Two-Layer Architecture Insight
*2026-04-25*

## 핵심 주장

Stage C (Objective 2) 모델은 사실 **2-layer 비선형 구조**다.
딥러닝과 구조적으로 동일하지만, 각 레이어의 설계 철학이 근본적으로 다르다.

---

## 구조 비교

### 일반 딥러닝
```
입력(가격)
    ↓ Layer 1: activation function (ReLU, sigmoid 등)
    ↓ Layer 2: linear combination
출력(예측)
```

### 우리 모델 (Stage C)
```
입력(가격)
    ↓ Layer 1: 전략 알고리즘 함수들 (MA crossover, Adaptive Band 등)
    ↓ Layer 2: linear combination + model selection module
출력(예측 수익률 → portfolio weight)
```

---

## Layer 1: 전략 알고리즘 (비선형 변환)

딥러닝의 activation function 대신 **사람이 설계한 전략 알고리즘**을 사용한다.

- Adaptive Band: 가격을 rolling band 기준으로 정규화한 스코어
- MA Crossover: 단기/장기 이동평균 차이
- Adaptive Volatility Band: 고가-저가-종가 범위 기반 변동성 레짐
- Fear-Greed Candle-Volume: 캔들 크기 + 거래량 스파이크 이벤트

이 함수들은 각각 가격 데이터의 **비선형 변환**이다.
파라미터(window, threshold 등)를 그리드 서치로 최적화하여 top-N을 선택하면,
4개 전략 × 10개 파라미터 셋 = **40개의 비선형 기저함수** 행렬 A가 만들어진다.

```
A ∈ ℝ^(n×40)  —  각 열이 하나의 비선형 basis function
```

---

## Layer 2: Linear Combination + Model Selection

딥러닝처럼 단순히 가중치를 학습하는 게 아니라,
**모델 선택 모듈**을 통해 가장 좋은 선형 결합기를 고른다.

- 후보 모델: OLS, Ridge, Lasso, ElasticNet
- 선택 기준: time-series CV-MSE (look-ahead 없음)
- 결과: 그 anchor 시점의 시장 국면에 가장 적합한 linear combiner 선택

```
ŷ_t = β₀ + Σⱼ βⱼ · A_{tj}
w_t = clip(ŷ_t / s, 0, 1)
```

## 왜 딥러닝 대신 이 구조인가

### 1. 데이터 규모 제약
- 선택 윈도우 1년 ≈ 252행
- 딥러닝 최소 요구 샘플: 파라미터 수 × 10~30
- 작은 MLP(2 hidden layer)만 해도 파라미터 ~1,800개 → 최소 18,000행 필요
- **252행으로는 완전 오버피팅**

### 2. 금융 시계열의 비정상성(non-stationarity)
- 시장은 사람들의 집단 심리와 트렌드로 움직임
- 2008 금융위기, 2020 코로나, 2022 인플레이션 — 각 시대마다 시장 구조가 달라짐
- 장기 데이터를 늘릴수록 **과거 국면이 현재 국면을 오염**시킴
- 1년 윈도우는 이 문제에 대한 의도적인 해법

### 3. 월별 앵커 업데이트가 핵심
```
딥러닝: 과거 전체를 한 번에 외우려 함
우리 모델: "지금 이 국면에서 유효한 lens는 무엇인가?" 를 매달 다시 질문
```

> *"The model does not learn the market — it learns which lens to look through,
>  and resets that lens every month."*

## 파라미터 랭킹과 신호의 위치

어떤 파라미터에 진짜 답이 있는가?

| 랭킹 구간 | 특성 | Forward 기여 가능성 |
|---|---|---|
| rank 1~2 | 선택 기간에 과최적화, 서로 높은 상관 | 강하지만 오버피팅 위험 |
| rank 3~6 | 강하면서 다양성 있음, 상관 낮음 | **가장 높음** |
| rank 7~10 | 약하지만 orthogonal 가능 | 낮지만 보완 역할 |
| rank 11+ | 대부분 노이즈 | 거의 없음 |

**rank 1이 아닌 rank 3~6 중간 영역**이 forward 성과에 가장 기여할 가능성이 높다.
rank-1과 rank-2는 파라미터 값이 거의 동일(중복)한 경우가 많아 Ridge/Lasso가 다운웨이팅하는 경향이 있다.

이것이 top-1만 쓰는 Stage A보다 top-10 전체를 쓰는 Stage C가 일관되게 강한 이유다.

- 실험 후보: top-20 (80 컬럼) — Lasso가 sparse 선택하므로 감당 가능
- top-40 (160 컬럼)은 252행 대비 고차원, 역효과 가능성

### 2. SINDy 이차항 확장
- Θ(A) = [sᵢ, sᵢ²] → 80 컬럼 (교차항 제외)
- 교차항 sᵢ·sⱼ = 780 컬럼 → 252행에 과도함
- 이차항만 추가하면 "강한 신호가 강세장에서 더 강해지는" 효과 포착 가능

### 3. 멀티에셋 노출도 상호보완성
- corr(w_GLD, w_BRKB) 낮을수록 자본 효율 높음
- 각 자산 신호가 서로 다른 시장 국면에서 BUY → idle 자본 순환 배치 가능

### 4. Layer 0 — 인간 매크로 인사이트 (미래 확장 철학)

현재 프레임워크는 2-layer지만, 궁극적으로는 3-layer 구조로 확장 예정:

```
Layer 0 (Human): 매크로 인사이트 → 섹터/자산 선택
                 매년 또는 적절한 시점에 인간 판단으로 결정
                 예) "올해는 방산주" → ITA, XAR
                     "조선주 사이클" → 한국 조선 ETF 또는 ADR
                     "기술주 랠리"  → QQQ
                          ↓
Layer 1 (Algo):  전략 알고리즘 → 비선형 피처 추출
                          ↓
Layer 2 (Model): Ridge/Lasso → 언제, 얼마나 투자할지
```

**핵심 철학**: 인간이 **무엇에** 투자할지 결정하고, 기계가 **언제, 얼마나** 결정하는 분업.
퀀트 프레임워크는 asset-agnostic이므로 config 파일 하나만 바꾸면 어떤 섹터에도 적용 가능.
